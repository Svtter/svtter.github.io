团队的技术选型

 发表于 January 30, 2021  |  1 分钟  |  43 个字  |  svtter

一个团队的技术选型对于团队发展非常重要。

技术选型会制约技术团队的发展。

读者

团队 Leader

编程语言

编程语言选型很大程度的决定了团队做事的范围和深度。

编程语言应该注意几个关键：

  • 语言热度
  • 对软件工程特征的支持性，如：软件复用，面向对象。
  • 语言性能

语言热度

语言热度意味着生态的发展速度。生态可能最初不佳，但是由于较高的语言热度，其周边会被迅速填充。 尤其是当优秀的工程师被这门编程语言吸引的时候，大量的优秀成果便会涌现。常见的例子便是 go 语言。 go 语言的书籍，例如go语言原本等，框架正如雨后春笋一样涌出。

相对的，当语言热度降低，其不完善性也会暴露出来。个人认为 Javascript 正处在这个境地。前几年node.js 破圈（前端圈），这两年除了deno以外，声音不大。

Python 虽然火爆，但是缺少高质量的工程师，其大部分支持者是初学者与科学家。这对语言的发展不是一个好事情。 明星开发者对语言的支持至关重要。缺少明星开发者，语言便会逊色。 例如，开源项目 requests , spider 等，极大的推进了 Python 在爬虫方面的优势。 大公司的支持，[Tensorflow][], [Pytorch][] 等也尤为重要。

基础设施

基础设施是决定团队技术选型的关键。如果硬件条件不允许，许多编程语言是做不了事情的。 常见的，如果生产机器只有512M内存，那么 Java 应该就不是一个很好的选择。
