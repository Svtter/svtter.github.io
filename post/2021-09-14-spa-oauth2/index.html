SPA 应用如何使用 OAuth2

 发表于 September 14, 2021  |  1 分钟  |  50 个字  |  svtter

SPA 是现在最为常见的前端开发技术。OAuth2 则是一个较为常见的鉴权框架。我们应该如何将 SPA 和 OAuth2 结合起来呢？

本文翻译自SPA。

Single-page apps (或者 基于浏览器的 app) 在完全加载 HTML 文件和 Javscript 文件后， 才会在网页中进行展示。由于浏览器可以访问整个前端的代码，因此，作为开发者的我们无法维护client secret的保密性，所以client secret将不会被用于这些应用程序。

SPA 加密的过程，和authorization code的流程基本相同，但是在最后一步，authorization code会交换一个 access token而不是使用一个client secret。

图 2 是一个示例。用户和浏览器进行交互，浏览器又直接向服务发起 API 请求。在从客户端下载 Javascript 和 HTML 代码后，浏览器会向服务发起一个请求。这种情况下，应用程序的服务器不会向服务发起请求（一次也不会）。这是由于一切过程都是在浏览器中处理的。

Authorization（鉴权）

authorization code 将会客户端当作一个临时的code来请求 access_token； 我们将会从 authorization server 获取 code，从而使得用户可以看到客户端正在请求哪些信息，从而允许，或者拒绝。

web flow的第一步，是从用户那里获取许可（authorization)。客户端通过创建一个authorization请求的链接来实现。
