Python GUI Technicals

 发表于 November 13, 2023  |  1 分钟  |  45 个字  |  svtter

GUI 指的是 Graphic User Interface。这篇文章是 Python 的桌面应用开发技术总结。WEB 应用我认为也是桌面应用的一种。

原本这个话题比较老旧了，但是近期出现了许多新的（或者说我近期看到了许多）Python 桌面开发技术，因此写这篇文章总结一下，给读者作为参考。

不同的框架解决的应用问题不同。在进行应用开发的时候，选择合适的框架很有必要。

技术名称 解决的问题 不足
PyQt 大型桌面应用开发，也能集成 WebView，相对全能 上手困难，见效慢
NiceGUI 偏向全栈的开发框架 全栈集成化程度高，进而导致定制化需要更多投入
Streamlit 用于机器学习应用的开发 如果不是机器学习应用，那么框架带来的助益很有限
Gradio 专注于机器学习应用的开发 -
Kivy 开发跨平台应用，除了桌面应用以外，iOS，Android 都能跑！ 上手难度大，需要学习如何使用其DSL定义界面
PySimpleGUI 快速跨平台桌面应用开发 界面相对朴素，很难开发出漂亮的界面
DearPyGui 小型 GUI 库，体积小 界面朴素

除了考虑框架解决的问题以外，开发者还要考虑可维护性，易上手程度。如果框架本身热度不高， 那么维护者可能停止维护，到时候就需要开发者投入更多的时间去处理框架本身的问题。

综合考虑以上特点，开发者可以选择自己喜欢的框架进行程序开发。

有时间继续更新。
