Dive Into Django and Webpack

 Posted on July 12, 2024  |  1 minutes  |  210 words  |  svtter  • Other languages: zh-cn

这是一篇没有完成的文章，后续还会进一步修改。

如何 Django 和 webpack 来提高开发效率？ 让我们从 django-webpack-loader 入手，来分析一下这个过程。我们可以看到，在django-webpack-loader项目中，可以通过如下的代码整合：

{% load render_bundle from webpack_loader %}

<html>
<head>
    {% render_bundle 'main' 'css' %}
</head>
</html>

在这个文件背后，Django Webpack Loader consumes a stats file generated by webpack-bundle-tracker and lets you use the generated bundles in Django.

  • webpack bundle tracker 可以将 webpack 编译过程中输出的 stats 文件整合到一个文件中。
  • stats 是什么呢？应该是 stats data，也就是 webpack 文档中提到的，可以用于提升编译性能的 module 统计信息。
  • bundles 是什么呢？bundle 可以看成 webpack 的集合体。html+js+css 打包后的产物。

也就是说，通过 django-webpack-loader，webpack 生成的 bundle 就可以在 django 中使用了。这是一个很酷的事情。

但是，我们也有一些疑惑。例如，对于一个传统的 react 项目，main.jsx 是入口，然后往往会 import 各种定义好的 Component，最后形成完整的前端页面。那么，对于一个直接对应 url 来书写 template 的框架，django 如何与前端的开发模式进行整合呢？如果你使用过 rails，那么就会思考，django 能否像 rails 一样，集成优秀的前端生态内容，但不改变现有的 MVC 开发模式。

如果你不想深入了解这个问题，可以直接跳转 django-react-boilerplate。优秀的开源贡献者们已经将这些集成好了。

遗憾的是，django-react-boilerplate 尚未像 rails 一样对 webpack 的小组件有充分的支持。django 目前还是以 app 为中心进行复用。在我看来，目前的模板项目，都是在项目管理工具的层面进行的简单整合。也就是说，即便是你将 frontend 代码单独拿出来，也是可以运行的。如果要做前后端分离的开发，这是一件好事。但是如果是 full-stack 的个人开发，我不认为这是一件好事情。

好处是，这一定程度上减少了开发者的成本。但是对于项目的模块化，还是有一个属于 django 的概念会更好。或许我们可以称之为 django components。更多的内容，可以直接阅读 [Dive into Django Components].

In Rails

让我们从用户的角度出发，看看 rails 是怎么做的。让我们从 rails-base-app入手。我们可以发现在 panel.html.erb 文件中，存在 vite_typescript_tag的标签。

这是否意味着 rails 嵌入前端代码的方式有所不同？是的。我们可以在 app 下的 javsscript folder 中找到对应 panel 页面的代码，也就是 panel.ts。

果真如此吗？

带着疑问我又一次访问了 django-webpack-loader，在 usage 章节，loader 明确说出自己是支持更加动态的引入 bundle 的。这里涉及到 entry 的概念，有更加具体的例子

  {% load render_bundle from webpack_loader %}
  <!DOCTYPE html>
  <html>
    <head>
      <meta charset="UTF-8">
      <title>Example</title>
    </head>

    <body>
      <div id="react-app"></div>
      {% render_bundle 'other' %}
      {% render_bundle 'main' %}
    </body>
  </html>

在这个例子中，我们可以看到 render-bundle 是有选择性的引入文件的。我认为这是一个核心的功能。

django webpack 还有更多的用法，可以从advanced usage中找到。

More discussion about Full-stack, MPV

将现代化前端（例如 react，vue） 与 django 进行混合开发的意义是什么？混合开发最大的意义是减少代码的书写量。如果进行前后端分离开发，我们需要定义接口，引入 swagger，openapi 等一系列的东西。但是，如果采用全栈框架， 就不需要了。我们可以用这个：

	  <div>
	    {{ form.as_p }}
	  </div>

我们在后端开发程序的时候，可以很轻松的定义好结构并且直接展示在前端页面。 当然，django component 也是一种复用方法。

How About Vite.js?

既然 django 能够集成 webpack，那么是否也可以集成更加受欢迎的 vite？是的。django-vite 就是这样做的。既然都是集成工具，用法应该是差不多的。
