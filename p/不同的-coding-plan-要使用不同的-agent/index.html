不同的 coding plan 要使用不同的 agent

本文探讨了Coding Agent领域的分裂趋势，指出大模型供应商正纷纷推出自有Harness，并提醒依赖模型能力的创业者需构建自身壁垒。

语速

这篇文章是我的暴论。

Coding Agent 的重新分裂是必然的，现有的 open source harness 将会面临新的危机。

首当其冲

Claude code 对 200K 模型不做 auto compact 兼容。

https://github.com/anthropics/claude-code/issues/46416

  • 之前从未承诺过兼容
  • 也就是说 GLM-5.1 在使用 claude code 的时候，可能直接打满上下文。你需要自己控制上下文长度。

Harness 自有化

vendor-lock 是作为用户尽可能避免的，但是是厂商所追求的。

  • MiniMax 推出自己的 Desktop Coding Agent；
  • Kimi 的 Kimi Code 目标都在此。
  • CodeX
  • Deepseek Harness 消息

之后只会越来越多的 LLM provider 做自己的 Harness。
