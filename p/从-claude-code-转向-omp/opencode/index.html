从 claude code 转向 omp/opencode

从 claude code 转向 omp/opencode 的原因和推荐

Jun 02, 2026 阅读时长: 1 分钟
语速

如果你不是在使用 claude 订阅，那么我建议你转向 omp（oh-my-pi）。

为什么

原因不复杂。目前按照 claude code 发展趋势和策略，claude 会朝向专用 agent 发展。每次 claude 更新都会遇到新的问题。Anthropic 接口虽然现在几个国内的厂商是兼容的，但是总归是有一定的时效。需要一定时间去 catch up 上游 Anthropic 的更改。

现状是 Anthropic 兼容 API 是厂商提供 coding agent/coding plan 的重要标准。但是实际上厂商也在转向。作为重要的组成部分，agent harness 一定要掌握在自己的手里。

opencode

如果喜欢 opencode，可以考虑使用 opencode。目前来说比较稳定，但是新功能开发不够快。

我之前写过很多 opencode 相关的文章，可以作为参考：

  • 高效省钱：我的 AI Agent 工作流选择 — OpenCode + Gemini 的高性价比工作流
  • 最近发现好用的 MCP 工具 — OpenCode/Claude Code 配套工具推荐
  • OpenCode 配置优化 — Token 和配置优化技巧
  • 探索 opencode 之后，我的一些思考 — 三个月使用回顾与思考
  • opencode-actions - 一个 coding review agent — 基于 OpenCode 的 GitHub Action

omp

但是我现在更加推荐 omp。

omp 对于 /goal、/loop 以及 workflow 的支持都非常快速。

omp commit 的迭代频率非常高，基本每天都在更新。
