Rewrite Code With Roo Code

使用 Roo Code 来尝试重写项目代码

Jul 12, 2025
阅读时长: 2 分钟

Roo Code 是 cline 的一个分支。

为了验证 roo code 是否好用，我尝试使用 roo code + deepseek v3 来重写 meterhub。

结果上来说，体验一般般。前期对于项目的构建等内容做的还不错。但是一旦涉及到部分具体的细节，很容易在一个问题上走不出去。

为了方便起见，下文我一律使用 roo code 来指代 (roo code + deepseek v3)。

结论

roo code 会卡在一些简单的问题上。例如使用 next auth 来设置邮箱链接登录，roo code 无法很好的完成任务。

这种问题会导致研发进度卡很久。这其实与这篇文章的结论比较相似。

研究显示：AI 并没有提升编程效率，它反而让你变慢了19%

刚刚，METR发布了一份关于AI 编程的研究报告，像一记重拳打在了整个AI编程工具行业的脸上。这不是什么小打小闹的实验室测试，而是16位资深开源开发者在他们自己维护的项目上进行的246个真实任务测试

微信公众号 · 发布日期

Context of Me

我对 nextjs 了解如何？我在编程过程中具有怎样的知识背景？

我本身对 nextjs 有一定的了解，但没有深入使用 nextauth 等工具的经验。因此，遇到问题我更加倾向于让 roo code 去解决问题。 我原本以为 roo code 重写的速度很快，而且很方便。正如这篇文章所说：

一个月重写三次代码库、三个月就换套写法！吴恩达：AI创业拼的是速度，代码不重要

近期，吴恩达 (Andrew Ng) 在 Y Combinator 发表了最新演讲，分享了自己的创业心得。他提出为创业公司成败的关键在于执行速度，执行速度比以往任何时候都更加重要，此外，他还详细阐述了创业公司应该如何提速。

InfoQ · 2025-07-11

但是实际体验下来，效果不行。可能也是因为:

  • 我本身只花了2天来去做这个事情。
  • 我对于 ts, nextjs, tailwind 技术栈没有很深入的了解

重写其实没有必要。最终我决定结束这个重写。

其他模型

再说说其他的模型的性能情况。

从实际使用上来看,

  • Deepseek-r1 的响应速度不太够。
  • 好用，但是 Claude 4 太贵。我只发起一次请求，掉了 3 块钱人民币。比我乘坐一次公交车还贵。
  • Kimi K2 不够聪明。生态兼容性也不好。我认为是一个比较初级的产品，落后其他的模型很多。
  • deepseek/deepseek-r1-distill-qwen-7b 速度够快，但是不够聪明

其他

吐槽一下：

  • 微信输入法会重复输入标点符号。我不得不在使用 vscode 的时候切换为搜狗输入法。
  • 火绒会截断不少使用 Admin 权限的安装程序，导致这些程序无法被被打开，但是火绒也不会有提示。
