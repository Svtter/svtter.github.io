我不喜欢火山引擎

Jun 02, 2026 阅读时长: 2 分钟
语速

准确来说，是我不喜欢这两家的 coding plan 产品。

翻转：MiniMax 给的实在是太多了。可在预算吃紧的前提下酌情使用。 第二次翻转：这篇文章原来叫做《我不喜欢 minimax 和火山引擎》。现在我去掉了题目中的 minimax。

MiniMax

一开始不喜欢 MiniMax 主要原因是 M2.7 的高分低能。之前感叹 M2.7 性能好，但是偶尔会出现幻觉已经随意改代码的情况。让人不敢用。

最近，但是最近发布的 M3，不仅仅打榜很好，实际效果也非常不错。

只不过在 omp 下：

  1. 工具调用频繁失败。这意味着 Benchmark 高，harness 没适配，也发挥不出。（近期已经修复）
  2. 存在费用问题。尽管 token plan 看起来便宜，但是每次执行工具都失败，也不顶用啊。（问题已经不存在）

omp（oh my pi）测试结果

BTW：这个问题 kimi k2 的时候还会出现，现在 kimi k2.6 早就没有这个问题了。

此外，还有文件修改出错的问题，我在 omp 上频繁遇到。

经过验证，我发现这是OMP的问题。

opencode

暂未出现问题。

但是幻觉问题还是比较严重。幻觉问题还是存在。

最后：不建议购买。我已经买了一个月，下个月不再续购。

火山引擎 - 字节 coding plan

火山在之前可以说是压根没法用。Claude Code 适配都没做好，就拿出来销售。让人感觉就是草台班子。

此前我大概买了三次，每次都是兴冲冲去，然后失望的退订。

什么，你说最近更新了？我是不会再上当了。
